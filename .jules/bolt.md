## 2024-05-24 - Avoid double matrix inversions
**Learning:** In statistical codebases dealing with inverse covariance matrices, it's common to compute an inverse matrix and pass it downstream. Sometimes, downstream functions redundantly invert this matrix back, leading to a mathematically equivalent identity (e.g., `inv(inv(A)) == A`), which severely degrades performance because matrix inversion is an O(N^3) operation. In `vuongtest.R`, `calcLambda` was unnecessarily inverting `A`, which was already the inverted form of the variance-covariance matrix (`tmpvc`).
**Action:** When working with matrices (like covariance and score matrices) in statistical packages, always track whether a matrix represents the original data or its inverse. Instead of blindly passing the inverted matrix and re-inverting it later when the original is needed, return both or pass the required form directly to avoid redundant O(N^3) recalculations.
